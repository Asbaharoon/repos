Steps To Evaluate the Test

 

1.      Test Set Up

1.1    New Test Set Up: User has to give the general information regarding the test.

a)      TestName:  User has give a unique name to the test. User can keep track of the test via this name.

b)      TestID: This is a a six-digit number, constructed by the instructor as per their institute requirement (internal exam or entrance exam) and their convenience. For example, they may use 2 digits for the Program code, 2-digits for the Course code, 1-digit for semester, 1-digit for test number for that course, or any other system they may want to adopt. This testId can be repeated in each session.

c)      Total Questions: Total questions in the test.

d)      Test Date:  It is the test conduct date

e)      Section: Select the number of sections in which user wants to divide the total questions. At least one section has to be selected.

 

 

 

 

1.2 Manage Test Set Up

User can update the information given by him before processing of the test.

If the correct Answers has been uploaded, only marks and Test Conduct date can be updated.

 

2.     Upload Correct Answers

 Correct Answers cannot be uploaded before the test conduct date.

User has to select the test conduct date, system filters the test name according to the time period selected by the user, then user has to select the test name for which he wants to upload the correct Answers.

 

Correct Answers can be uploaded in two ways:

a)     Via scanned sheet

               User can fill the correct answers in the OMR sheet and then upload the sheet.

 

 

 

 

b)    Manually via UI

 

 

Before finally submitting the correct answers, system once displays all the correct answers to the user for verification.

 

 

 

 

3.     Upload Response Sheet

          In order to upload the response sheets, user has to select the test date and test name and then upload the sheets in a zipped folder.

 

 

 

 

4.     Process test

                   In order to process the test user has to select the test conduct date and the test name that he wants to process.

 

 

After processing of the test, the system displays the total number of sheets in the zipped folder, number of sheets processed by the system and the number of sheets rejected by the system.

 

5.     Result

5.1     View Marks

It displays the section wise detail of marks obtained by the candidate. On clicking the Candidate ID, response sheet of the candidate, responses of the candidate read by the system and the correct answer of each question.

 

In case, if candidate commits some mistake while filling the candidate Id, -1 will be displayed against that candidate Id. Though the marks of such candidate will be computed by the system and candidate can be identified by the name written on the response sheet.

The system also provides the utility to transfer the result into excel sheet.

 

5.2 Manage Result

        By default, result will be displayed for seven days after processing of the sheets. However, instructor can extend this period maximum to a month after processing of the sheets.

 

 

 

5.3Log

There are two log tables:.

a)     Error Log: It records all the errors encounter  in the response sheets while processing.

 

b)    Test Log: It displays the number of sheets processed and rejected by the system.

 

6.     Misprinted Questions                       

                                    There may be some misprinted questions in the question paper. Instructor may either opt to provide grace marks for that question or discard that question (exclude from evaluation).  Such questions can be entered only

 

6.1 Insert Question

                                 This UI is provided to insert the misprinted question and set its status.

 

 

 

 

 

6.2 Delete Question

                                This UI is provided to delete the misprinted question inserted earlier.

 

 

7.     Download Template

                                   User is provided with the utility to download OMR sheet template.
