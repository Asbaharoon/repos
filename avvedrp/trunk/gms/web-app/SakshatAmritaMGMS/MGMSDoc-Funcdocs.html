MGMS - Functional Documentation

The functional specifications for the various functionalities have been arrived at after many thorough discussions with the users, developers and the implementation team.  The system will be constanlty evolving and as such the documents provided below are ‘living documents’ . The specification will change, continuously, as the project proceeds. More details will be added and the existing information edited as the requirements, scenarios and use cases evolve in the course of the project. 

Functional objective

Multisite GMS provides for day to day project management. Moreover, because the system is web-based, can be assessed from anywhere, provides a medium for a collaborative working environment for internal/external experts and reviewers.

In today’s business environment, Grant-Giving Organizations need to be ever more accountable to its officials and Government Departments and indeed those who are accepted and rejected for various grants. MGMS can track each and every application there by making accountability, reporting and governance much easier. The system can also be easily integrated in future with leading financial systems to better keep track of spending.

Highlights of MGMS

  • Easily accessible from anywhere, for MGMS being a web deployed application.
  • Permission based security allows full or partial access to screen and data depending on the user’s needs.
  • Reports based on multiple criteria.

Post - Project Award Functionalities The features are involved with creation of the received project, assigning assets and team members, allocating funds to different accounts as per the terms & conditions, possible sub allocation of parts of the project to other institutions (sub projects), project tracking etc.

  • Asset purchase
  • Human Resource management
  • Site administrator creation
  • Project creation
  • Expense entry
  • Expense approval request
  • Grant receipt
  • Head-wise allocation
  • Project - home
  • Project tracking
  • Project search
  • Project sub allocation
  • Utilization report submission

 

Pre-award Functionalities Pre-award features enable users to create grant notifications, manage them within the institution, call for proposals, evaluate them and finally select the grantee. There is also provision to create and send proposals electronically across instiutions which use the MGMS

  • Proposal Application
  • Grant notification
  • Notification Creation
  • Internal Proposal Evaluation
  • Proposal prequalification
  • Proposal Management

 

System Administration As the name suggests, these documents explain the features which are mainly used by the system administrators of the member institutions of MGMS. Tasks include registering the institution in MGMS, adding new users, assigning roles & privileges, monitoring usage, generating MIS reports etc.

  • Institution creation
  • Settings
  • User management
  • Login
