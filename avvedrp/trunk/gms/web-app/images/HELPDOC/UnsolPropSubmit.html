MHRD Sakshat NME ICT Proposal Submission

Currently the provision for applying to MHRD has been implemented.

  1. Anyone may apply using this form to the particular granting agency. It is not required that the applicant’s institution be already registered in MGMS. 
  2. There is option to select institution for applying to. It is expected that different member institutions of MGMS will have different application forms.
  3. The Sakshat NME ICT form has 7 parts – where, in the 6th page – there is mandatory uploading of the PI CV and the DPR. The 7th page captures the overall summary of the entire filled up form.
  4. On submitting the page 7, the preview is displayed. From here edit feature is enabled.
  5. First page – filling up is mandatory and on saving this page, a proposal is officially created in the GMS system.
  6. A control ID is generated, also on saving the first page and a mail sent to the applicant’s email id confirming it. The control ID is also displayed on all subsequent pages of the application form.
  7. On final submission of proposal, an alert e-mail is sent to all members of the default approval authority.
  8. Versioning is provided to the extend that – the first time a form is submitted it is assigned version “V0” and after that each the same form is opened using the control ID, the version number is incremented – for e.g. V2, V3, V7 etc

MHRD Sakshat NME ICT Application form:

On clicking Upload Proposal in the Log in page, the screen appears where in all those grant scheme notifications by the MGMS user institutions are displayed for which proposals are solicited from the general public. On clicking View for a Notification, the details are displayed. This window can be closed by clicking “x” in the top right hand corner. 

To apply for a particular scheme, click “Apply” for the associated notification. In the resulting screen, there is provision to enter a control ID to revise an already submitted proposal (a saved draft form may be retreived) or the user may start a new form by clicking on New Form.

On clicking New Form, as per current client requirement, the 7-page MHRD Sakshat form opens up for any notification.

The first page of the 7-part form opens up. On this page all fields are mandatory.

Please note that the Project Category field on this page displays the  proposal categories saved for the particular funding agency by the site administrator. For e.g. "Infrastructure" is a project category for NME ICT. So the MHRD (which is the grantor) site administrator should create "Infrastructure" proposal category in the Settings module..

On clicking Next in page 1, a control ID is generated which will be used for all further reference to this proposal at any later time. An e-mail is sent to the applicant with the control id value.

From second page onwards, the Control ID generated is displayed at the top of every page and also the current version of the document being edited (“V0” now since this is the first draft).

In all rich text editors, it should be possible to include tables (10-15 rows, 5-6 columns, images, graphs etc).

In pages 2, 3, 4 & 5, fields are not mandatory but in each case, the user can move to the next page only after clicking Next on the current page.

In page 6, upload of CV and DPR are mandatory.

In page 7, all fields are mandatory. On clicking Submit in this page, the preview of the details filled in so far, is displayed.

On clicking Edit for any section in the Preview page, the corresponding page opens up for editing. In that page, after editing, on clicking Update, the preview page returns displaying the changed data.

Now on clicking Finish, the proposal is officially submitted for evaluation as version V0.

On trying to open the proposal again by submitting the control id, the system displays message that it is already submitted and the user is provided with 3 options  (Links) –

Details provided by you – opens the Preview page
Enter revised proposal – opens Revision of Proposal screen where the user has to select “I agree” option to open the Preview – Edit page
Upload Proposal – opens new proposal form to start again

On clicking Enter revised proposal link, screen opens where the user has to select “I agree” option and click Submit to open the Preview – Edit page.

After making the necessary changes and updating the proposal, the user can click Finish. The version number of proposal changes, each time a proposal is revised in this manner and the version change is updated for the reviewers screens also..
On submission of the proposal, it is available for evaluating in the log-ins of all reviewers who are part of the institution’s default approval authority.

In the screens, clicking on Cancel button opens the MGMS log-in page.
