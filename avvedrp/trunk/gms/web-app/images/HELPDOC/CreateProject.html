Project Creation

The site admin can create new projects for the institution. The key associated activities are:

  • Saving the project details (which may be updated any number of times) - project name, code, type, Principal Investigator, Project Start & End Dates.
  • A project can have only a single Principal Investigator at a time.
  • The user may set a project as active/inactive as appropriate in case of future changes in the programme. Thus system can retain all the past data for a currently inactive project but not display or involve it in the current activities & reports, once the “Inactive” selection is made.

On the left menu panel, the administrator clicks on Projects menu on the left navigation bar. It expands to display the sub menu – Create Project.
On clicking Create Project, the page opens up where the main details are captured for saving the project - project name, code, project type, PI & Project Start & End Dates.

On clicking Create, the user is returned to the Project List with a message on top confirming successful creation of the new project and the new project name listed out in the Project List.

On clicking the project name, the Project Info tab is opened to display the default Project Home page. Here the basic details captured are displayed. Further, on mouse roll over of the Project Info tab, further sub menus are displayed - namely Co-PI Addition & Department Addition.

 

Add Co-PI

  • The institution can assign any number of Co-PIs to the project after project creation.
  • The details saved are – name and role (co-PI).
  • The user may add/delete co-PI as appropriate in case of future changes in the project team structure. Thus system retains all the past data for a currently closed project.

Add department.

This is an option for the institution to assign the created project to particular departments.

  • The detail saved is the department name, selected from the list of departments made available.
  • Any number of departments may be associated with a project.
  • Editing/updating may be done any number of times

Grant Allocation

  • While creating a project, a grant allocation is created automatically with a default amount of zero.
  • The site admin can update with the actual grant amount the institution is allocating for the created project (self-fund). Here, the Institution is the owner of the project.
  • The details saved are – amount allocated, and any remarks from the user.
  • Any number of grant allocations can be made during the project period by the institution
  • An allocation can never be deleted.
  • An allocation cannot be edited after sub-project allocation

For the institution site admin (& privileged users), there is provision to view all its current projects in a single list (Project List), from which the  user may navigate to any project’s home page.

In case of a scenario where an institution has been sub allocated a project, the details will be captured by the grantor institution and the project is listed automatically in the Project List of the grantee. The grantee, in this case, has no privilege to modify the project details such as name, code, start & end dates, PI etc.
