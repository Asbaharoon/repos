NME ICT - Final Proposal Evaluation

 
Currently, the final proposal evaluation is based on the NMEICT model of evaluation.

Proposal evaluation includes the following features:

  1. Calculation by which the average score is arrived at, for each proposal and the score display.
  2. A proposal may be resubmitted any number of times and a parameter V<x> will indicate the corresponding version number (x= 0, 1, 2, 3 etc). The reviewer can review the latest proposal in the system.
  3. On final submission of proposal, an alert e-mail is sent to all members of the default approval authority (of the sponsoring institution)
  4. Proposal Administrator role is assigned to the person who will capture the evaluation details – parameters and criteria to judge , scoring pattern, questionnaire etc
  5. All scores to be viewable by Proposal Admin. Final scores will be viewable by reviewers at any point of time and each can edit own scores
  6.  Proposal status request– user is prompted to enter the control number, once the submit button is clicked, the complete details of the submitted proposal is displayed.
  7. Option is provided for the Site administrator & Proposal administrator in which each reviewer’s score for every question in the evaluation questionnaire is displayed.

Initial Administrative Functions

Site Admin: Add New Role – ROLE_PROPOSALADMIN

If not already saved in the application, this role needs to be saved and assigned to the user who needs to receive all evaluations from all the reviewers. Only a single person in an institute may be assigned this role. (In future the role may be assigned to each grant scheme for which proposals are called.).

Site Admin – Default approval authority

Default approval authority must be assigned for the institution. In this way, proposals submitted for all notifications will be directly sent for review to the members of the default approval authority. There can be only one default authority per institution.

On the left menu panel, the administrator clicks on Approval Authority menu. It expands to display the sub menu – Add Approval Authority
On clicking Add Approval Authority, the page below opens up. Here each group of approving persons at the institution is registered. Further, for each approval authority, the members are also added.

The personal details are: Name of the group, group e-mail ID and Institution. Further, the Approve Mandatory field decides if the particular group’s approval is mandatory and it is set to “Y” (Yes) by default. View All is set to “Y” (Yes) by default which indicates that the members of this group are privileged to view all proposals even if they don’t get to review them.
When Default Authority field is set “Y”,   it means that all proposals submitted for all notifications in that institution will be sent for review to the members of that approval authority. Each proposal is not separately assigned.   Currently when a default approval authority already exists, a different approval authority cannot also be made default for the same institution.
In the Approval Authority List, for each line item (approval authority) , clicking on Add Members opens the page below, where members can be added to/removed from each Approval authority. The existing users are displayed in the table on the right  from which selection can be made. In the Approval Authority List, for each line item (approval authority), clicking on Edit, opens the page where any modification can be made.

Site Admin – Notification creation for a grant scheme

In Call for Proposal sub menu of Projects Menu, the site administrator creates a new notification. Please note that in this document we are explaining the process for publically solicited proposals. So the notifications created are assigned the attribute “Public – Yes”. This means that the notification gets displayed in a Notification List made available outside the MGMS application as well.

Proposal Admin - Proposal evaluation Pre-settings

As mentioned before, the proposal administrator is in charge of various managing the evaluation process. For each grant notification created and published by an institution, the proposal admin assigns the specific evaluation parameters; ratings required and prepare the questionnaire for use by the reviewers.

After logging in, the user clicks on menu Proposal evaluation – pre settings, and sub menu Evaluation scale. Here the user can create a new evaluation scale as well as view all the scales created in this institution till now. Edit option is also provided to modify any existing scale’s name

After naming the scale, the user clicks Evaluation Scale Options where the standard scores can be listed. For e.g. if the reviewer selects the option “Good” as per the Likert scale, for a particular answer , then the asnswer has scored 3 marks. 
In case the user wants to score in number format, a new scale can be defined where Scale option “10” can be assigned Scale option index “10” .

Once the scoring pattern is decided, the user clicks on Evaluation Question, to view the criteria or questions that are already saved for the institution in the application, with associated scoring pattern. The user may add any further question to this list for judging the proposals submitted for a particular notification.

The user clicks on sub menu Assign Evaluation Questions for Notification. Here, for a selected notification, any number of questions from the existing list can be added to the review questionnaire.

Applying against the notification

An applicant applies for the particular scheme by clicking “Apply” for the notification published in the Notification List. Currently, the same 7 page form of IGNOU Sakshat has been made available for all institutions and notifications.

On final submission of the proposal, it is available for evaluating in the log-ins of all reviewers who are part of the institution’s default approval authority.

Reviewer – Proposal List

The reviewers who belong to the default approval authority can see the proposal as a line item in their Proposal List. On clicking View, the preview page of the proposal is opened up.

On clicking Review Proposal for a particular proposal, the screen opens up where both the proposal as well as the review form are displayed. Top portion of the screen displays the entire proposal in the preview format, and the bottom half displays the questionnaire to be used for evaluation of the proposal. Maximize button has been provided for both sections in case the user wishes to view the complete sections separately.

The questionnaire displayed here is the one which was created by the Proposal Administrator initially (Proposal evaluation – pre settings).
Please note that there maybe n number of reviewers and so at any point of time, a score will be displayed averaging the scores that have been assigned till then by one or more reviewers.

Each reviewer puts the score as deemed fit for the particular proposal and may enter any comments. It is mandatory that all questions be answered. Clicking Save will just save the details as draft. On clicking Submit, the scores are officially submitted into the system for calculation of the final score for the proposal.

The final rating of the proposal is calculated as the average score. Currently, all reviewers can see the final score and the number of votes cast which resulted in that final score. “Votes cast” is the number of reviewers who have reviewed the proposal.
The Proposal administrator and Site administrator can also view the same list of scored proposals in the Projects menu - Call for proposal sub-menu. In the Notification List, clicking View in Proposals Received column for a particular notification will open the page displaying the proposals that were submitted against the notification and their final scores at any point of time.

Option is provided for the Site admin & Proposal admin to view the review scores/details entered by each reviewer against proposals. Clicking on Proposal Review link for a proposal, will open a screen where each reviewer’s score for every question in the evaluation questionnaire is displayed. Mouse roll-over the question number will display the question as a tooltip.
