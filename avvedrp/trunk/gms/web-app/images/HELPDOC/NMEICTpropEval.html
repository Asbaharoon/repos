NME ICT - Final Proposal Evaluation

 
Proposal evaluation including the following features:

  1. Calculation by which the average score is arrived at, for each proposal and the score display.
  2. A proposal may be resubmitted any number of times and a parameter V<x> will indicate the corresponding version number (x= 1, 2, 3 etc). The reviewer can review the latest proposal in the system.
  3. On final submission of proposal, an alert e-mail is sent to all members of the default approval authority (of the sponsoring institution) – currently a simple mail format is used
  4. Proposal Admin role is assigned to the person who will capture the evaluation details – parameters and criteria to judge , scoring pattern, questionnaire etc
  5. All scores to be viewable by Proposal Admin. Final scores will be viewable by reviewers at any point of time and each can edit own scores
  6.  Proposal status request– user is prompted to enter the control number, once the submit button is clicked, the complete details of the proposal submitted is displayed:

Initial Administrative Functions

Site Admin: Add New Role – ROLE_PROPOSALADMIN

If not already saved in the application, this role needs to be saved and assigned to the user who needs to receive all evaluations from all the reviewers. Only a single person in an institute may be assigned this role. (In future the role may be assigned to each grant scheme for which proposals are called.).

Site Admin – Default approval authority

Default approval authority must be assigned for the institution. In this way, proposals submitted for all notifications will be directly sent for review to the members of the default approval authority. There can be only one default authority per institution.

Site Admin – Notification creation for a grant scheme

In Call for Proposal sub menu of Projects Menu, the site administrator creates a new notification. Please note that in this document we are explaining the process for publically solicited proposals. So the notifications created are assigned the attribute “Public – Yes”. This means that the notification gets displayed in a Notification List made available outside the MGMS application as well.

Proposal Admin - Proposal evaluation Pre-settings

As mentioned before, the proposal administrator is in charge of various managing the evaluation process. For each grant notification created and published by an institution, the proposal admin assigns the specific evaluation parameters; ratings required and prepare the questionnaire for use by the reviewers.

After logging in, the user clicks on menu Proposal evaluation – pre settings, and sub menu Eval scale. Here the user can create a new evaluation scale as well as view all the scales created in this institution till now. Edit option is also provided to modify any existing scale’s name

After naming the scale, the user clicks Eval Scale Options where the standard scores can be listed. For e.g. if the reviewer selects the option “Good” as per the Likert scale shown in the image below, for a particular answer , then the asnswer has scored 3 marks. 
In case the user wants to score in number format, a new scale can be defined where Scale option “10” can be assigned Scale option index “10” .

Once the scoring pattern is decided, the user clicks on Eval item, to add the criteria or questions for judging the proposals submitted for a particular notification.
As can be seen below, 7 questions are assigned to the notification code “NMEICT” and the Eval Scale used will be “Likert scale”.

An applicant applies for the particular scheme by clicking “Apply” for the notification. Currently, the same 7 page form of IGNOU Sakshat has been made available for all institutions and notifications.

On final submission of the proposal, it is available for evaluating in the log-ins of all reviewers who are part of the institution’s default approval authority.

Reviewer – Proposal List

The reviewers who belong to the default approval authority can see the proposal as a line item in their Proposal List. On clicking View, the preview page of the proposal is opened up.

On clicking Review Proposal for a particular proposal, the screen opens up where both the proposal as well as the review form are displayed. Top portion of the screen displays the entire proposal in the preview format, and the bottom half displays the questionnaire to be used for evaluation of the proposal. Maximize button has been provided for both sections in case the user wishes to view the complete sections separately.
This was the questionnaire created by the proposal administrator initially (Proposal evaluation – pre settings).
Please note that there maybe n number of reviewers and so at any point of time, a score will be displayed averaging the scores that have been assigned till then by one or more reviewers.

Each reviewer puts the score as deemed fit for the particular proposal and may enter any comments. It is mandatory that all questions be answered. Clicking Save will just save the details as draft. On clicking Submit, the scores are officially submitted into the system for calculation of the final score for the proposal.

The final rating of the proposal is calculated as the average score. Currently, all reviewers can see the final score and the number of votes cast which resulted in that final score. “Votes cast” is the number of reviewers who have reviewed the proposal.
The Proposal admin and Site admin can also view the same list of scored proposals in the Projects menu, Call for proposal sub-menu. In the Notification List, clicking View in Proposal column for a particular notification will open the page displaying the proposals that were submitted against the notification and their final scores at any point of time.
