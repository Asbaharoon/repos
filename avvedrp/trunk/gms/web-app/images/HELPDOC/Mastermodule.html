Settings Module

Through the Settings module, the user can define his/institution specific options for the following entities. Once these lists are created in the GMS, the particular institution user can select from these options, where ever required.

  • Grant Agency: Institutions which act as grantors to the user’s institution.
  • Account Head: The various expense account categories and their sub categories as used by the user’s finance department. For e.g., manpower, material purchase, asset etc
  • Sub account head: within each account head, sub account may also be created. For e.g manpower account head may have sub account heads – salary, wages, professional fees etc
  • Grant Period: The specific terms/sequence format used by the particular institution to indicate a project grant period. For e.g  “2001-02” or ‘Jan10-Aug10”
  • Project Type: To categorize own projects further, for e,g to distinguish between “State funded”, “Central funded”
  • Attachment Type: The business nature of the uploaded document – for e.g proposal, RFP, sanction letter, notification etc. Attachment types should be saved for specific document types so that they get displayed as options in associated modules. The document types currently pre-set in MGMS are - Invoice, receipt, proposal, notification and project. For example, attachment types saved for document type Project will populate the Attachment Type drop down menu in Sub project's attachment upload screen.(Similarly, attachment types saved for document types Receipt, Invoice etc will be displayed in the associated Expense Request screens.)
  • PI: The list of all Principal Investigators at the Institution. (the PI will get activated only when they click on the activation link which has sent to their email id, entered during the creation of PI.
  • Role: The various job profiles associated with the Institution and each requiring a specifi set of access privileges and permissions. For e.g., Manager, Scientist, Clerk etc
  • Employee Designation: As the name suggests, the various job titles given by the Institution to its employees. For e.g. Senior Scientist II, Asst Manager
  • Salary Component: The various salary components as per the salary structure of the institution. For e.g. Basic, HRA, LTA etc

Options created in Settings Module are editable and may also be inactivated, when required. Thus system can retain all the past data for a currently inactive user-defined option but not display or involve it in the current activities & reports, once the “Inactive” selection is made.
