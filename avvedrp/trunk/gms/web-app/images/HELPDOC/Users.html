Users, Roles & Privileges

The site administrator adds all the other users for an institution. Then the users are assigned their roles which automatically assigns certain privileges to them and further, if required, the users are provided the privileges to specific projects.

To access these features, log in as the sie admin and on the left navigation bar, click Users to expand the menu. The following sub meus are then available.

Add user

Using this functionality, the institution site admin can create any number of new users for the GMS.

  • The details saved for this profile are – Full name, password, email and role in the institution
  • A confirmation email with the activation link and login details will be sent to the email address entered during registration.  
  • Account will get activated by clicking the activation link provided in the email.
  • There is an option for the user to change the login password. If the user forgets the password, the system will reset the password and send it to the email entered by the user based on request.
  • A user accessing the GMS using the login created by an institution’s site admin, will have privileges and permissions corresponding to the role chosen and the access will also be only to that particular institution’s GMS.

 

Project Permission

Here, a new user may be given access to a number of projects all at once, by simply selecting the projects.
Once they are selected, the user can view the list of projects he is privileged to view, when he logs into the GMS.

A few ideal advisable permissions are given below:

    As a PI or CoPI, a user should have visibility of all the project for which
      • User is PI or CoPI, and all the sub_projects under these projects
      • Should not have access to other projects under the same Institute
      • Should not have access to other projects under the same department
      • Should not have access to main_project if only PI/CoPI for sub_project
      • Should not have access to the sister sub_projects for which not PI/CoPI

      Access Permission

      This functionality enables the site admin to create and customize sets of privileges/privilege groups for each role created for the institution.

       The advantage is that, when a role is chosen for a new user, he is automatically & immediately given permissions and rights corresponding to the role. 

      Role Privileges

      This is a further provision to assign all of the privileges in the GMS to a specific role (aside from that of site admin) , using a single click.
