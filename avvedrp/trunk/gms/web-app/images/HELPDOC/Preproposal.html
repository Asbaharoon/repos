Pre proposal Submission

Initial Administrative Functions

Adding new users - reviewers:

A new role “Reviewer” has been added to GMS. The reviewers can be added by the Administrator through Add User page and the role can also be assigned here.The reviewers created are displayed in the User List.

The privileges for the role “reviewer can be saved in the Access Permission page.

Creating approval Authorities:

On the left menu panel, the administrator clicks on Approval Authority menu. It expands to display the sub menu – Add Approval Authority
On clicking Add Approval Authority, the page below opens up. Here each group of approving persons at the institution is registered. Further, for each approval authority, the members are also added.

The fields bordered in red are mandatory.
The personal details are: Name of the group, group e-mail ID and Institution. Further, the Approve Mandatory field decides if the particular group’s approval is mandatory and it is set to “Y” (Yes) by default. View All is set to “Y” (Yes) by default which indicates that the members of this group are privileged to view all proposals even if they don’t get to review them. Setting a group inactive here will result in removing the group from all user options. It is then deemed to be non-existing.
In the Approval Authority List, for each line item (approval authority) , clicking on Add Members opens the page where members can be added to/removed from each Approval authority. The existing reviewers are displayed in the table on the right (faculty) from which selection can be made.

In the Approval Authority List , for each line item (approval authority), clicking on Edit, opens the page where any modification can be made.

Upload Pre-proposal Application form

The admin logs in the system and on the left menu panel, clicks Proposal Authority map. When the menu expands, the admin clicks on Upload Proposal Form . Here, the admin can browse and upload the form used by the institution to capture details of the proposal which constitutes the pre-proposal.

Add Proposal Category

The Admin has to save the various proposal categories in the GMS application for users (staff) to select from while creating their proposals. Examples for category are Research, e-content, infrastructure, network etc. This is done in the Masters module, sub menu – Proposal Category

Submission of Pre-proposal by staff members

The staff includes those users who submit pre-proposals /proposals for evaluation. The Staff logs into the application to create and submit pre-proposals to own institution for review. After logging in, user clicks Pre proposal Menu on the left panel and then clicks Add New Proposal.

On clicking Create, the institution’s pre-proposal application form is opened for the Staff to fill up the pre-proposal details:

After filling in the details, on clicking Create, the user is given options to – save the document as draft or to submit to authorities for review.

The Pre proposal status list displays the specific Staff‘s list of pre-proposals which have been submitted so far and their status. ‘Reviewing” status indicates that the pre-proposal submitted is under review. If the user had chosen to save as draft then this field would have displayed “Saved”.

Edit button allows editing and re-submitting of the pre-proposals.
When in “Reviewing” status, clicking on “Reviewing” will open the page, where the details of the reviewing process for the particular pre proposal is given

Once the admin assigns the pre-proposal to the approval authorities, sets the start order, restart order etc, those details are shown here. Before this is done the page simply displays message - "No Details"
This process is described below:

Proposal Authority Mapping

The admin logs in the system and on the left menu panel, clicks Proposal Authority map. When the menu expands, the admin clicks on Assign proposal to approval authority sub menu.

The details can also be edited at a later time.

Once the admin saves the above details, the pre-proposal details get populated for each member of the approval authorities to whom the specific pre-proposal was assigned.

Pre – proposal Review

The reviewer log in, clicks Pre proposal menu on the left and when the menu expands, clicks Pre proposal Approval.  The list of proposals to be reviewed by this reviewer at various evaluation levels is displayed here.

If, for a particular pre-proposal, this reviewer belongs to the approval authority of the first level (Approve order – 1), then in the Approve /Reject column the corresponding status will be “Approve/Reject”. On clicking this status label, the pre-proposal is opened and the reviewer is provided with 3 buttons – Accept, Need more info, Reject.
But if the reviewer belongs to any other authority level, then till the pre-proposal passes through the other levels before his own, the status in the Approve /Reject column will be “View” for this reviewer (and all others at this authority level). On clicking View, the pre-proposal form can be opened and viewed only.

The reviewer has been provided with a text area to input comments.
Once a suitable status (Approved) is selected, the line item disappears from the specific reviewer’s list and the line item populates the reviewers’ lists at the next authority level.
If “Need more info” status is selected then the line item returns to the Staff’s list.  The Staff can resubmit after adding the required details and the chain now follows the Process Restart order.
