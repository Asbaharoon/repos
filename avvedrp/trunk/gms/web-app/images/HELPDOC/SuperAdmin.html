Super Administrator

Features

When GMS is made available for public (to download and install) the details of this super admin credentials will be given as part of a Read Me.txt.

Default user with username and password.

Username – “admin”
Password – “admin123”

User name and password are provided in the seed data which is loaded during the implementation of GMS.

Once GMS is implemented and prepared with sufficient data by the super admin, all the institutions may register at the link provided to them or the super admin can register these institutions in its network.

Institution

As the super admin of GMS, you can:

    Create  institutions

      View and  also edit  all the Institutions details which are registered in the MGMS system

      User Accounts

      As the super admin of GMS, you can:

      • Create user for each institution
      • View all the users among the institutions
      • Edit user details including password

      Role Privileges

      As the super admin of GMS, you can:

      • Create role privileges

      Master Settings

      As the super admin of GMS, you can:

      • Enter master details which are common to all  institutions in that GMS setup:
        • Account Head
        • Grand Period
        • Project type
        • Attachment Type
        • Role

      So, these options are not provided to the institution site administrators, from now on.
